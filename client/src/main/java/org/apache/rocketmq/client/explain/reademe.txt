消息发送者(Producer):
RocketMQ支持三种发送方式：同步(sync),异步(async),单向(oneway)
同步：发送者向MQ执行发送消息API时，同步等待，直到消息服务器返回发送结果
异步：发送者向MQ执行发送消息API时，指定消息发送成功的回调函数，然后调用发送消息的API之后立即返回，
消息发送者线程不阻塞，直到运行结束，消息发送成功或者失败的回调函数在另一个线程中执行。
单向：消息发送者向MQ执行发送消息API时，直接返回，不等待消息服务器的结果，也不注册回调函数，简单说，
就是只管发送消息不在乎消息是否发送成功是否存储在消息服务器上。

消息拉取基本流程：
1.消息拉取客户端封装消息拉取请求
拉取入口方法：DefaultMQPushConsumerImpl#pullMessage()
2.消息服务器查找并返回消息
消息服务端消息处理类入口：org.apache.rocketmq.broker.processor.PullMessageProcessor#processRequest()
3.消息拉取客户端处理返回的消息
